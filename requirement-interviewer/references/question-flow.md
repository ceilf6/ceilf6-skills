# Question Flow

Ask questions in phases. Do not skip ahead unless earlier phases are already answered clearly.

## Phase 1: Objective

Goal: understand what the user wants to achieve.

Ask about:

- target outcome
- primary user or audience
- success condition

Example prompts:

- What should exist when this is done?
- Who is the user of this feature or page?
- What would make you consider the work accepted?

## Phase 2: Scope

Goal: define what is in scope and what is out of scope.

Ask about:

- required screens, flows, or components
- exclusions
- must-have versus nice-to-have items

## Phase 3: Constraints

Goal: capture implementation boundaries.

Ask about:

- framework or stack constraints
- design system constraints
- performance, accessibility, or compliance requirements
- approval-sensitive limitations

## Phase 4: Delivery

Goal: define how the result should be verified and handed off.

Ask about:

- acceptance criteria
- testing expectations
- deadline or sequencing constraints
- expected final output format
