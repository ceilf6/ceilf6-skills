# Answer Completeness

Use this file to decide whether to follow up or move forward.

## A complete answer

Treat an answer as complete when it:

- directly resolves the current question
- removes the main ambiguity
- is specific enough to affect implementation or acceptance

## A partial answer

Treat an answer as partial when it:

- answers only part of the question
- uses vague language like "something nice" or "modern"
- leaves multiple valid interpretations open

When an answer is partial:

- ask one focused follow-up
- narrow the choice set
- avoid introducing new topics

## User-delegated decisions

When the user says "you decide" or similar:

- pick a sensible default
- state it explicitly as an assumption
- continue only if the assumption is low risk

## Stop conditions

Stop asking when:

- all mandatory gates are satisfied
- further questions would not change the plan materially
- the remaining gaps are low risk and can be labeled as assumptions
